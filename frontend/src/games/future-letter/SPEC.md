# Спецификация игры «Письмо из будущего»

## Контекст
Игра — часть SPA «Girls (8 Марта)». Пользователь проходит мини‑игры, чтобы получить сертификат. Эта игра — интерактивная «панель управления» письмами из будущего.

## Цели
- Дать пользователю выбор параметров письма (месяц, горизонт времени, температура сценария).
- Сгенерировать письмо на основе параметров и показать его в виде «листа».
- Визуализировать доставку письма через анимацию совы.
- Ограничить число писем до 3 за сессию.

## UX/Flow
1. Пользователь открывает игру и видит панель управления.
2. Выбирает:
   - Месяц (12 ближайших месяцев).
   - Горизонт: ближайшие 12 месяцев, +5 лет, +50 лет.
   - Температуру (реализм → постапок).
3. Нажимает «Получить письмо».
4. Появляется сова с письмом. По клику на сову письмо открывается.
5. По клику на письмо оно сворачивается и пользователь возвращается в панель выбора.
6. После каждого письма доступна кнопка «Выставить РДС на сову для получения нового письма».
7. Максимум 3 письма за сессию (по достижении лимита — блокировка выдачи новых писем).

## Текст письма
### 12 ближайших месяцев (сезонность и корпоративные циклы)
Текст письма должен включать нюансы времени года и ритма финансовой организации:
- Январь: переходный период после праздников, мягкий вход.
- Февраль: стабилизация, подготовка к планированию.
- Март: закрытие суперспринта Q1.
- Апрель: премии, запуск планирования Q2.
- Май: стабильная нагрузка, реализация инициатив.
- Июнь: закрытие суперспринта Q2.
- Июль: планирование Q3, более спокойный летний ритм.
- Август: поддержание темпа, сезонная просадка.
- Сентябрь: закрытие суперспринта Q3, сбор метрик.
- Октябрь: планирование Q4, разгон перед пиком.
- Ноябрь: высокий сезон, рост нагрузки.
- Декабрь: максимальная нагрузка, годовое закрытие.

### Горизонт +5 лет
- Заголовок письма от имени председателя правления «одного синего банка».
- Обращение по имени пользователя.
- В письме указывать дату, из которой оно отправлено (текущая дата + 5 лет).
- Отсылки к известным фильмам — без прямых названий.

### Горизонт +50 лет
- Заголовок письма от имени комбинированного киберорганизма председателя правления «одного синего банка».
- Обращение по имени пользователя.
- Дата письма: текущая дата + 50 лет.
- Отсылки к известным фильмам — без прямых названий.

## Температура сценария
Бегунок 0..100:
- 0–40: реалистичные и позитивные сценарии.
- 41–70: фантастические, но оптимистичные.
- 71–100: позитивные, но постапокалиптические (апокалипсис с надеждой).

## Данные пользователя
- Имя пользователя сохраняется при логине в `localStorage` (`girl_name`).
- В письме использовать имя для персонального обращения.

## Технические требования
- Реализация в `frontend/src/games/future-letter/`.
- Стили в отдельном CSS‑файле (например, `FutureLetterGame.css`).
- Сова реализуется встроенным SVG/CSS, без внешних ассетов.
- Кликовый сценарий:
  - Клик по сове открывает письмо.
  - Клик по письму закрывает и возвращает к панели выбора.
- Лимит писем хранить в `sessionStorage` (например, `future_letter_count`).

## Кнопка повторного письма
- После прочтения письма показывать кнопку:
  - Текст: «Выставить РДС на сову для получения нового письма».
- Кнопка возвращает к панели выбора и разрешает следующую выдачу письма (если лимит не достигнут).

## Критерии готовности
- Линтер проходит: `npm run lint`.
- Ручные проверки:
  - Все 12 месяцев корректно формируются от текущей даты.
  - Сезонные тексты соответствуют месяцам.
  - Заголовки +5/+50 корректны, есть дата отправки.
  - Анимация совы и открытие письма работают.
  - Клик по письму возвращает к панели.
  - Лимит 3 письма соблюдается.
